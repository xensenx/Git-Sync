Quick Start

Already have a GitHub account and know your way around? You'll be up and running in under two minutes.

On this page

  1. Install the Plugin
  2. Personal Access Token
  3. Configure Settings

Step 1

Install the Plugin

The plugin is not yet on the Community Plugins browser, so install manually:

  1. Download main.js, manifest.json, and styles.css from the GitHub repo.
  2. Create the folder <vault>/.obsidian/plugins/direct-github-sync/
  3. Drop all three files into that folder.
  4. In Obsidian → Settings → Community Plugins, disable Safe Mode if needed, then enable Direct GitHub Sync.
💡
Tip

On mobile, use a file manager app (e.g. Files on iOS, solid explorer on Android) to navigate to your vault folder and create the plugin directory.

Step 2

Personal Access Token

DGS authenticates via a GitHub PAT with repo scope (or public_repo for public repos only).

  1. Go to github.com/settings/tokens
  2. Click Generate new token (classic)
  3. Set a note, choose expiry, tick repo scope
  4. Click Generate token — copy it immediately, it won't show again
⚠️
Store it safely

Treat your PAT like a password. Keep it in a password manager. If it leaks, revoke it immediately from GitHub Settings.

Step 3

Configure Settings

Open Settings → Direct GitHub Sync and fill in:

Required Fields 
GitHub PAT      →  ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
GitHub Username →  your-github-username
Repository Name →  your-vault-repo
Branch          →  main  (or master, or whatever you use)

Optional but recommended:

Optional Fields
Device Name     →  MacBook / Phone / etc.
Auto-Sync       →  Enable + set idle interval (minutes)
Sync on Startup →  ✓ Enabled
Ignore .obsidian→  ✓ Enabled (default)
Step 4

First Sync

Click the sync icon in the left ribbon, or open the Command Palette (Ctrl/Cmd + P) and run Direct GitHub Sync: Sync Now.

On first sync:

  • If the repo is empty — your vault is pushed as the initial commit.
  • If the repo has existing files — they're pulled down and merged with your local vault.
  • If the repo doesn't exist yet — create it on GitHub first (can be private), then sync.
Reference

Settings Reference

GitHub PAT

Required

Your Personal Access Token. Must have repo scope for private repos, public_repo for public only.

Username

Required

Your exact GitHub username, case-sensitive. Used to construct API endpoints.

Repository

Required

The repository name (not the full URL). E.g. my-obsidian-vault

Branch

Default: main

Which branch to sync to. Defaults to main. Change to master or any custom branch as needed.

Ignored Paths

Optional

One rule per line. Supports exact paths, directory prefixes (private/), and glob wildcards (*.canvas). .obsidian/ has its own toggle above.

Device Name

Optional

Appears in commit messages as Vault sync from MacBook: 22 Apr 2026 at 14:30. Helps identify which device made a change.

Auto Sync + Interval

Optional

When enabled, DGS fires a sync after the vault has been idle for the specified number of minutes. Defaults to 5 minutes.

Sync on Startup

Default: on

Automatically pulls from remote when Obsidian launches. Keeps your vault current before you start editing.

Concurrency

Default: 5

How many files to upload/download simultaneously. Reduce to 2–3 on slower connections or if you hit rate limits.
