Authentication

Authentication Errors

401 Authentication failed — your PAT is invalid or expired

Your Personal Access Token is wrong, expired, or was deleted.

Fix: Go to github.com/settings/tokens, check if your token exists and hasn't expired. If it has, generate a new one (see the Beginners Guide) and paste it into the plugin settings.

💡

Double-check for accidental spaces before or after the token when pasting. Clear the field and re-paste carefully.

403 Access forbidden — PAT may lack 'repo' scope

Your token exists but doesn't have permission to access this repository.

Fix: Go to your token settings on GitHub and check the scopes. You need the repo scope for private repositories. If you only checked public_repo, you won't be able to sync a private repo. Generate a new token with full repo scope.

403 GitHub rate limit exceeded

You've made too many requests to GitHub's API in a short time.

Fix: Wait 5–10 minutes and try again. GitHub resets rate limits per hour. If this happens regularly, try reducing the Concurrency setting to 2–3 in the plugin settings, or increase the auto-sync idle interval.

Repository

Repository Errors

404 Repository "username/repo" not found

The plugin can't find the repository you specified.

Most likely causes:

  • Typo in the Repository Name field — check exact capitalisation
  • Typo in the Username field
  • The repo doesn't exist yet — create it on GitHub first
  • The repo is private and your PAT lacks repo scope
404 Branch "main" not found in "username/repo"

The plugin found your repo but the branch name doesn't match.

Fix: On GitHub, look at your repo. The branch name is shown in a dropdown near the top-left (it may say master instead of main). Update the Branch field in plugin settings to match exactly.

409 Repository is empty — will be initialised automatically

This is not actually an error — it's informational. Your repo exists but has no commits yet.

What happens: DGS will automatically push all your local files as the initial commit. Just wait a moment and it will complete.

Sync

Sync Errors

Error Network error — check your internet connection

DGS couldn't reach GitHub's servers.

Fix: Check your internet connection and try again. If you're on mobile, ensure Obsidian has network permissions. DGS automatically retries failed requests up to 3 times before reporting this error.

422 GitHub rejected the request (422)

A file you tried to sync was rejected by GitHub.

Common causes:

  • A file is larger than 100 MB (GitHub's hard limit per file)
  • A filename contains characters GitHub doesn't allow
  • You're trying to push to a protected branch

Fix: Large files should be excluded via the Ignored Paths setting. Add the problematic file's path to the ignore list.

Warning GitHub tree truncated (repo >100k files)

Your repo is very large (over 100,000 files) and GitHub isn't returning all of them in a single API call.

Fix: This is an edge case most users won't hit. Consider using the Ignored Paths feature to exclude large directories (e.g. an attachments/ folder) from sync.

Mobile

Mobile Issues

iOS Plugin doesn't appear in the Community Plugins list

The plugin folder wasn't created correctly or Obsidian hasn't detected it yet.

Fix:

  • Use the Files app to verify the folder is at <vault>/.obsidian/plugins/direct-github-sync/
  • Make sure all three files (main.js, manifest.json, styles.css) are inside that folder
  • Close and fully reopen Obsidian (swipe it away from the app switcher)
  • If using iCloud, wait for files to fully sync down before reopening Obsidian
Mobile Sync works on desktop but fails on mobile

Settings may not have been configured on the mobile device, or the PAT has expired.

Fix: Settings are per-device. Open Settings → Direct GitHub Sync on your phone and re-enter your PAT, username, repo, and branch. These are stored locally and not synced between devices for security.

Gotchas

Common Mistakes

❌

Using the repo URL instead of just the name

The Repository field should contain only the repo name, e.g. my-vault, not https://github.com/username/my-vault.

❌

PAT doesn't have 'repo' scope

If your repo is private, you must check the full repo checkbox when generating your token, not just public_repo.

❌

Editing on two devices simultaneously without syncing first

Always sync before you start working on a new device. If you edited the same note on two devices without syncing, DGS's three-way merge will try to reconcile them, but the result may not be what you expect.

❌

Plugin folder name has wrong capitalisation or extra characters

The folder must be named exactly direct-github-sync — all lowercase, with hyphens. No spaces, no underscores, no extra letters.

❌

Initialising the repo with a README

If you created the repo with a README, it already has a commit. DGS will pull the README down on first sync. This isn't a crash — but it means you'll have a stray README in your vault. Delete it and re-sync.
