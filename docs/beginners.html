Beginner's Guide

Never used GitHub? Never heard of Git? That's perfectly fine. This guide starts from absolute zero and walks you through every single step.

🌱
You don't need to know Git

This plugin communicates directly with GitHub's web API. No terminal, no git install, no command line. If you can use Obsidian, you can use this.

Background

What is GitHub?

GitHub is a website that stores files in the cloud, with a superpower: it keeps a full history of every change you ever make. Think of it like Google Drive, but for files that track their own edit history automatically.

For us, GitHub is simply a place to store a backup of your Obsidian vault. Every time you sync, your notes are saved safely in the cloud. You can access them from any device, and if something goes wrong, you can go back to any previous version.

Step 1

Create a GitHub Account

If you already have one, skip ahead to Step 2.

1

Go to github.com

Open github.com in your browser and click Sign up.
2

Enter your details

Provide your email, choose a password, and pick a username. Your username will appear in your vault's sync history, so pick something you're comfortable with.
3

Verify your email

GitHub will send a verification code to your email. Enter it to confirm your account.
4

You're in

Skip the onboarding survey if you like. You're now on your GitHub dashboard.
Step 2

Create a Repository

A repository (or "repo") is just a folder on GitHub. You'll create one to hold your vault.

1

New repository

Click the + icon in the top-right corner of GitHub and select New repository.
2

Name your repo

Pick a name like my-obsidian-vault or notes. No spaces — use hyphens. Write this down, you'll need it later.
3

Set visibility

Choose Private if your notes are personal (recommended). Choose Public only if you want the world to see your notes.
4

Create it

Leave all other options as-is and click Create repository. Leave the repo empty — DGS will populate it on first sync.
⚠️
Don't add a README

When creating the repo, don't initialise it with a README, .gitignore, or license. Keep it completely empty. DGS will handle the first commit.

Step 3

Create a Personal Access Token

A Personal Access Token (PAT) is like a special password that lets the plugin write to your GitHub repository. It's more secure than your actual password because you can revoke it anytime.

1

Open Token Settings

Click your profile picture → Settings → scroll all the way down to Developer settings → Personal access tokens → Tokens (classic).
2

Generate new token

Click Generate new token (classic). GitHub may ask for your password again — this is normal.
3

Fill in the details

Note: Write something like "Obsidian DGS"
Expiration: Choose 90 days or "No expiration" (your preference)
Scopes: Check the repo checkbox (this gives full repo access — required for private repos)
4

Copy your token immediately

After clicking Generate token, you'll see a long string starting with ghp_. Copy it right now — GitHub will never show it again. Save it somewhere safe, like a notes app or password manager.
🔐
Never share your PAT

This token has write access to your repos. Don't paste it in public, don't send it to anyone, and don't commit it to a file. If it leaks, go to GitHub → Settings → Developer Settings → Tokens and delete it immediately.

Step 4

Install the Plugin

🖥️

Desktop

Navigate to your vault folder on your computer. You can find it by opening Obsidian → gear icon → Files & Links → look for "Vault folder location".

📱

Mobile

On iOS use the Files app; on Android use a file manager like Solid Explorer. Your Obsidian vault is usually in the Documents or iCloud Drive folder.

1

Download the files

Go to the GitHub repository. Download these three files: main.js, manifest.json, styles.css
2

Create the plugin folder

Inside your vault folder, navigate to .obsidian → plugins. Create a new folder named exactly: direct-github-sync
3

Place the files

Move all three downloaded files into direct-github-sync/
4

Enable in Obsidian

Open Obsidian → Settings (gear icon) → Community Plugins → if prompted, turn off Safe Mode → find Direct GitHub Sync in the list and toggle it on.
💡
Don't see the plugin?

Try restarting Obsidian after placing the files. If it still doesn't appear, double-check that all three files are inside a folder named exactly direct-github-sync (no typos, lowercase).

Step 5

Configure the Plugin

Go to Settings → Direct GitHub Sync. You'll see several input fields. Fill them in:

GitHub PAT

Paste the long token you copied in Step 3. It starts with ghp_.

GitHub Username

Your GitHub username exactly as it appears on your profile. It's in the top-right of GitHub after you log in.

Repository Name

The name of the repo you created in Step 2. Just the name — not the full URL. E.g. my-obsidian-vault

Branch

Leave this as main unless you specifically changed it when creating the repo.

Device Name (optional)

Something like MacBook or iPhone. This shows up in your sync history so you know which device made each change.

✅
Recommended settings for beginners

Turn on Sync on Startup and Auto Sync with a 5-minute idle interval. This means your notes sync automatically — you never have to think about it.

Step 6

Your First Sync

Close the Settings panel. You should see a small sync icon (two circular arrows) in the left sidebar of Obsidian. Click it.

You'll see a status message at the top of the screen. On first sync it might take a minute if you have many notes. After that, only changed files are synced so it's very fast.

ℹ️
What just happened?

DGS compared your local vault with the (empty) GitHub repo and uploaded all your files. If you now go to your GitHub repository page, you'll see all your notes there as files. You can even read them on the GitHub website!

Day-to-day

Ongoing Use

After initial setup, using DGS is completely invisible if you enabled Auto Sync. Your notes back up automatically.

🔄

Manual Sync

Click the sync ribbon icon, or use the Command Palette (Ctrl/Cmd + P) → search "sync".

📲

Sync Across Devices

Install the plugin on your phone or another computer, configure it with the same settings, and your vault will stay in sync automatically.

⚠️
When using multiple devices

Always sync before you start editing on a new device. This pulls the latest changes from GitHub so you're working with the most current version of your notes.
